# Product

## Register

product

## Users

Single administrator (the owner of the machine running Easy Access hub) who manages 2-5 remote machines on a mixed LAN + internet topology. The admin is technically literate — they set this up themselves. Primary context: seated at a desk, often working across multiple tasks, needing quick orientation on server status and fast file navigation. Sessions are deliberate, not casual.

## Product Purpose

Easy Access is a self-hosted remote server and PC data management platform. The admin can browse, upload, download, and preview files on remote machines via a central web dashboard. Success: the admin can glance at the dashboard, know which machines are up, and navigate to any file on any server in under 10 seconds.

## Brand Personality

Precise, reliable, calm. Three words: controlled, sharp, trustworthy. The tool should feel like it was made by an engineer who cares — not a startup trying to impress. No fluff. Every element earns its space.

## Anti-references

- Avoid SaaS-gradient purple glow aesthetic (Vercel dark mode clone)
- Avoid "hacker terminal" green-on-black (this is a polished admin tool, not a CTF challenge)
- Avoid warm cream/beige tones (wrong register, wrong user)
- No glassmorphism by default (purposeful only, not decorative)
- No hero-metric stat template (big number + gradient accent = banned)

## Design Principles

1. **Information first.** Every design choice must make the data easier to read, not harder. Ornamentation that competes with content is wrong.
2. **One moment of calm.** The dashboard is where the admin comes to resolve issues. The UI should reduce cognitive load, not add excitement.
3. **State clarity above all.** Online/offline, loading/loaded, selected/unselected — these states must be unmistakable at a glance.
4. **Density earns its cost.** Compact layouts are welcome when the data warrants it. White space is not a luxury; it's a tool for separation.
5. **Precision in motion.** Transitions only where they clarify state changes. No decorative animations.

## Accessibility & Inclusion

WCAG AA minimum. All interactive elements keyboard-navigable. Status indicators never use color alone (always accompanied by text or icon). Reduced-motion respected for all animations.
